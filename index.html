HW #1. Шпаргалка для користування GIT

  • git config --global user.name "Roman" - встановити ім'я користувача
  • git config --global user.email roman@gmail.com - встановити email користувача
  • git init - створити новий репозиторій
  • git status - представлена команда показує стан файлів в робочій директорії та індексі
  • git add "file" or . - додати всі змінені файли (to stage)
  • git commit - створити новий коміт
  • git log - показати історію комітів
  • git chekout - повернутись до останнього коміту
  • git branch - показати всі гілки
  • git push - виконує мерж локальних змін на віддаленний репозиторій
  • git pull - працює подібно до команди git push але виконує мерж змін з віддаленого репозиторію до локального
  • git merge - використовується для злиття однієї або декількох гілок в поточну
  • git clone - використовується для клонування локального або віддаленого репозиторію
  • .gitignore - дана команда слугує для вказування в ній файлів і папок які необхідно приховати від системи контролю версій git
  • git revert - представлена команда створює "антикомміт" для вказаного коміта, таким чином відміняє зміни, що були внесені в ньому.
  • git reset - в соновному дана команда використовується для відміни змін. Також змінює показник HEAD та, опціонально, стан індекса.
  • git rebase - Основна її мета "перетягувати" коммітов з одного місця в інше. Також за допомогою неї можна позбавлятися від непотрібних коммітів, склеювати одні і міняти порядок інших.
  • git cherry-pick - використовується для перенесення окремих коммітов з одного місця сховища в інше, зазвичай між гілками розробки та обслуговування.
  • git stash - дана команда призначена для того, щоб помістити поточні зміни, які Ви виконали в файлах, в окреме сховище, і повернути файли до вихідного стану.
